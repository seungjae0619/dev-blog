## **✅ 들어가며**

웹 브라우저를 사용하다 보면 우리가 작성한 HTML, CSS, JavaScript 코드가 화면에 표시되는 것을 볼 수 있다.
이 과정에서 웹 브라우저는 코드를 해석하고, 이를 시각적으로 표현하는데, 이 과정을 '렌더링'이라고 한다.
렌더링은 웹 브라우저가 사용자에게 콘텐츠를 보여주는 핵심적인 과정으로 , 웹 개발자라면 이 과정에 대해 이해하는 것이 중요하다.

## **✅ 렌더링의 과정**

웹 브라우저의 렌더링 과정은 크게 다음과 같은 단계로 이루어진다:

![img](https://velog.velcdn.com/images/jaehooo13/post/8800fdd7-296f-4914-adde-afe28b20dc92/image.png)

1. **HTML 파싱**: 브라우저는 서버로부터 받은 HTML 문서를 파싱하여 DOM(Document Object Model) 트리를 생성한다.
2. **CSS 파싱**: 브라우저는 HTML 문서에 포함된 CSS를 파싱하여 CSSOM(CSS Object Model) 트리를 생성한다.
3. **렌더 트리 생성**: DOM 트리와 CSSOM 트리를 결합하여 렌더 트리를 생성한다. 이 트리는 실제로 화면에 표시될 요소들로 구성된다.
4. **레이아웃 계산**: 렌더 트리를 기반으로 각 요소의 위치와 크기를 계산한다.
5. **페인팅**: 계산된 레이아웃 정보를 바탕으로 각 요소를 화면에 그린다.
6. **합성**: 여러 레이어로 나누어진 요소들을 하나의 화면으로 합성한다.

### **🤔 HTML 파싱**

HTML 파싱이란 문서를 문자열 그대로 읽어서 컴퓨터가 다룰 수 있는 트리로 바꾸는 과정이다. 브라우저는 HTML 텍스트를
화면에 그리지 못하므로 파싱을 해줘야한다.

```html
<html>
  <body>
    <h1>Hello</h1>
  </body>
</html>
```

위에 html 코드는 그냥 문자이므로 브라우저는 이걸 읽고 DOM 트리를 만든다.

DOM 트리 구조:

- Document
  - html (Element Node)
    - body (Element Node)
      - h1 (Element Node)
        - "Hello" (Text Node)

이런 트리 구조를 생성하는 과정이 바로 파싱이다.

DOM에 대한 내용은 다른 블로그에서 더 자세히 다루도록 하겠다.

## **✅ 렌더링 최적화**

렌더링 과정은 복잡하고 시간이 많이 소요될 수 있기 때문에, 웹 개발자들은 렌더링 최적화를 통해 사용자 경험을
향상시키려고 노력한다. 다음은 몇 가지 렌더링 최적화 기법이다:

- **리소스 최적화**: 이미지, CSS, JavaScript 파일의 크기를 줄이고, 필요한 리소스만 로드하여 렌더링
  속도를 향상시킨다.
- **비동기 로딩**: JavaScript 파일을 비동기로 로드하여 렌더링 차단을 방지한다.
- **CSS 최적화**: 불필요한 CSS 규칙을 제거하고, 중요한 스타일을 인라인으로 작성하여 렌더링 속도를 개선한다.
- **레이아웃 쓰래싱 방지**: 자바스크립트에서 DOM 조작을 최소화하여 레이아웃 계산 횟수를 줄인다.
- **브라우저 캐싱 활용**: 자주 사용하는 리소스를 캐싱하여 재방문 시 빠르게 로드할 수 있도록 한다.
