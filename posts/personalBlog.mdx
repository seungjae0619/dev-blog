## Next JS로 블로그 만들기

문제1: dynamic routing 을 사용해서 posts 폴더에 있는 mdx 파일들을 가져오고 싶음

nextjs 공식 홈페이지를 찾아 보니 동적 라우팅을 통한 mdx 파일 가져오는 방법이 있다!

```js
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>,
}) {
  const { slug } = await params;
  const { default: Post } = await import(`@/content/${slug}.mdx`);

  return <Post />;
}

export function generateStaticParams() {
  return [{ slug: "welcome" }, { slug: "about" }];
}

export const dynamicParams = false;
```

위에 코드 설명
동적 라우팅을 통해 params를 가져오므로 파라미터 정의
(gpt는 Promise가 필요 없다고 하는데 없으면 오류남 (멍청한 새기) -> 나중에 확인하기)

```js
const { default: Post } = await import(`@/content/${slug}.mdx`);
```

위 코드는 import를 동적으로 할 때 사용하는 문법이다. (처음 봄)
기존에 모듈을 가져올 때는 최상단에 모듈을 import를 해서 가져오는 정적 import만을 사용함
여기서 default는 export default를 할 때의 default를 말한다.
이렇게 하면 /blog/hello 라는 url 로 진입하면 hello.mdx를 불러올 수 있다.

generateStaticParams 는 빌드 타임에 정적 파라미터를 nextjs에 알려주는 역할이다
